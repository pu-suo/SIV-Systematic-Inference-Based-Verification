CONSTANTS vs ENTITIES — CRITICAL DISTINCTION
  • constants  → Proper nouns, named individuals, specific events/places.
                 MUST go in the constants array, NEVER in entities.
                 Use the camelCase form of the name as the ID.
                 Examples: Bonnie → id="bonnie", Lana Wilson → id="lanaWilson",
                           After Tiller → id="afterTiller", Harvard → id="harvard"
  • entities   → Quantified things: indefinite nouns ("a car"), universal ("all kids"),
                 definite generics ("the tree", "the region").
                 Use e1, e2, … as IDs.

ENTITY TYPE (entities only)
  • existential  → indefinite: "a car", "some bears", "the tree"
  • universal    → quantified over all: "all kids", "every student", "no bird"

FACT ARITY (implicit from len(args)):
  • 1-arg → unary predicate (type or property): Tall(e1), Queen(bonnie)
  • 2-arg → binary relation: Directed(lanaWilson, afterTiller), Likes(e1, e2)
  • 3-arg → ternary relation: Gave(bonnie, e1, e2)

2. USE COMPOUND ANALYSIS — the "COMPOUND ANALYSIS" block (when present) shows

3. MACRO TEMPLATE — choose the single best-matching template:
   • universal_affirmative   (TYPE A):  All P are Q       → ∀x(P(x) → Q(x))

   • ground_positive                    P(c) or P(c) ∧ Q(c)
