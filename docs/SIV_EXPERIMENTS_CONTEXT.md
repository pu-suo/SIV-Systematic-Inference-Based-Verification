# SIV — Unified Experiments Context

**Purpose.** Single reference for what every experiment in the SIV paper is, how
it's run, what numbers it produces, what it proves, and how it ladders into the
overall claim. Designed to be handed to a future Claude session to load full
context fast.

**Status as of writing:** Stages 1–4 complete (parser + test-suite generation
+ perturbation detection + graded-correlation re-score all passing). C1, C2,
and the D re-framing remain open. Three weeks to EMNLP target.

---

## 1. The overall claim

**SIV is a prover-grounded, graded, per-aspect diagnostic metric for NL→FOL
translation faithfulness.** Built on gold-derived test suites generated by a
deterministic parser; scored via Vampire entailment checks.

The paper argues two things:

1. **Gradedness.** SIV's scalar score tracks structural error severity in a way
   binary equivalence metrics cannot, and surface-form metrics (BLEU,
   BERTScore) don't track at all. *Headline number: ρ = 0.8543.*

2. **Per-aspect actionability.** SIV's per-probe failure trace carries
   error-type-specific information, and that information produces a concrete
   downstream win when an LLM uses it for self-correction. *Headline number:
   TBD from C2.*

Plus one supporting contribution and one artifact:

3. **Parity on the basic case.** SIV reaches binary-equivalence-metric
   detection rates on meaning-altering perturbations.

4. **Augmented FOLIO + 30 hand-corrected gold + parser** released as
   supplementary artifact.

What the paper is **not**: a translator, a benchmark, a sweeping framework,
or "reference-free" (that claim was retired during the Approach C pivot).

### Claim → experiment map

| Claim | Experiment | Status |
|-------|------------|--------|
| Gradedness (headline) | B | ✅ Done, ρ = 0.8543 |
| Per-aspect diagnostic carries structured info (intrinsic) | C1 | ⏳ Re-score needed |
| Per-aspect diagnostic is actionable (extrinsic, headline) | C2 | ⏳ Pilots, then run |
| Parity on basic case (supporting) | A | ✅ Done, Stage 5 verifies port |
| Gold corruption is detectable (supporting + artifact) | D | ⏳ Re-frame + small run |

The argument arc: **A** establishes "we're not worse on the easy case." **B**
establishes "we're better on the graded case." **C1** establishes "the trace
carries the right kind of information." **C2** establishes "that information
is actionable downstream." **D** establishes "we surface broken references
that other metrics swallow."

C2 is the load-bearing extrinsic result. B is the load-bearing intrinsic
result. Without both, the paper is weaker than its title.

---

## 2. Hard rules that govern all experiments

These survived the v1→v2 pivot and apply throughout.

- **Round-trip Vampire equivalence is a hard gate** for parser correctness.
  No exceptions, including for cleanup items.
- **No LLM in the test-suite-generation path.** This is what the pivot was
  for. C2 uses LLMs as *subjects* of the experiment, not as components of
  the metric.
- **Don't touch cached exp1/exp2/exp3 outputs.** Re-scores produce new files;
  v1 numbers stay locked.
- **Selection rules for hand-curated data are a priori, deterministic, and
  released as artifacts.** No post-hoc filtering.
- **ρ acceptance for Exp B: ≥0.81 expected, 0.78–0.81 investigate, <0.78
  stop.** Don't slide to 0.75 or 0.78.
- **Pre-register before running.** Every experiment with a statistical test
  has its analysis plan frozen in markdown before data is seen.

## 3. Experiment A — Operator-level perturbation detection (parity)

### What it is

Apply contrastive perturbation operators to gold FOL; check whether SIV's
test suites flag the perturbed version below threshold. Compare detection
rates against binary-equivalence metrics, BLEU, BERTScore.

### Method

- **Inputs:** FOLIO aligned-subset premises, gold FOL parsed via v2
  deterministic parser.
- **Operators:** B_arg_swap, B_negation_drop, B_restrictor_add,
  B_restrictor_drop, B_quantifier_swap, B_scope_flip, D_random.
- **Score function:** v2 test suite (positive sub-entailment probes +
  contrastive probes), Vampire-checked.
- **Detection rule:** perturbed formula scored against the suite generated
  from gold; "detected" if score below threshold.

### Numbers (already obtained)

Stage 3 results, v2 suites:

| Operator | Applicable | Detected | Rate | Avg perturbed recall |
|----------|-----------|----------|------|----------------------|
| B_arg_swap | 122 | 120 | 98.4% | 0.130 |
| B_restrictor_add | 95 | 95 | 100% | 0.004 |
| B_quantifier_swap | 120 | 106 | 88.3% | 0.120 |
| B_restrictor_drop | 38 | 0 | 0% | 1.000 |
| B_scope_flip | 3 | 0 | 0% | 1.000 |
| **Overall** | | | **84.9%** | |

Stage 5 (next): re-score v1 Exp A perturbation candidates with v2 suites.
Acceptance: ±5pp on B_arg_swap, B_negation_drop, D_random vs. v1 numbers.
Drift on B_restrictor_drop and B_scope_flip is the architectural blind spot
and is documented, not a failure.

### What it proves

SIV reaches parity with binary-equivalence metrics on detecting
meaning-altering perturbations. BLEU/BERTScore don't track logic; binary
equivalence does, but lacks gradedness. SIV does both.

### What it doesn't prove

That SIV is *better* than binary equivalence on this task. It isn't, by
design — both succeed when meaning is altered. The point of A is "we don't
lose the easy case."

### Architectural blind spot (disclosed limitation)

Recall-based positive probes can't penalize over-generalization.
B_restrictor_drop produces a logically stronger formula that still entails
all sub-tests. This is correct behavior under recall scoring; contrastive
probes catch it (precision-flavor). The 0% detection on these operators is
not a bug, it's a known property of the design and gets a paragraph in
limitations.

## 4. Experiment B — Graded correlation with structural error severity (HEADLINE INTRINSIC)

### What it is

For each premise, generate a graded family of candidate translations
(gold / overstrong / partial / overweak / gibberish) where the rank order
reflects intended structural fidelity. Score each candidate with v2 suites.
Measure Spearman ρ between intended rank and SIV score across premises.

### Method

- **Candidates:** 5 types per premise (gold, overstrong, partial,
  overweak, gibberish). v1 had 35 premises with full candidate sets.
- **Scoring:** v2 test suite recall (gold-derived).
- **Statistic:** Spearman ρ across premises, paired permutation test for
  significance.
- **Acceptance gate:** ρ ≥ 0.81. Pre-registered.

### Numbers (locked)

| Metric | ρ | 95% CI | n |
|--------|---|--------|---|
| v1 locked | 0.8563 | [0.8236, 0.8811] | 35 |
| **v2 (a) full set with regeneration** | **0.8543** | **[0.8217, 0.8779]** | **33** |
| v2 (b) full set without regeneration | 0.7797 | [0.6579, 0.8674] | 35 |
| v2 (c) equivalent-only subset | 0.8583 | [0.8225, 0.8801] | 22 |

(a) is the headline. (b) and (c) are robustness views.

Mean v2 recall by candidate type, set (a):

| Type | v1 | v2 | Δ |
|------|-----|-----|---|
| gold | 1.00 | 1.00 | 0.00 |
| overstrong | 1.00 | 1.00 | 0.00 |
| partial | 0.36 | 0.37 | +0.01 |
| overweak | 0.07 | 0.05 | -0.02 |
| gibberish | 0.03 | 0.03 | 0.00 |

### The principled filter (must explain in paper)

Of 35 v1 premises, Vampire bidirectional-equivalence check found 22 where
v1-LLM-canonical FOL ↔ v2-parsed-gold and 13 where it didn't. The 13
non-equivalent cases had their candidates regenerated against v2 gold using
the same protocol; 9 got all 4 non-gold types verified, 4 got 1–2 types
missing. ρ computed for premises with ≥3 non-gold types (n=33).

Two named cases drove the (b) ρ drop:

- **P1077:** v1 LLM invented `TypeOfLeukemia` predicate and restructured
  consequent. "Gibberish" candidate was logically stronger than v2 gold,
  so v2 correctly scored it recall=1.0.
- **P0621:** Gold uses constant `michael`; v1 LLM used `michaelODonnell`.
  Symbol alignment couldn't bridge at threshold=0.6.

These are candidate/gold mismatches inherited from v1 LLM canonicalization,
not v2 suite-quality regressions. The fix (regeneration against v2 gold)
preserves the headline.

### What it proves

SIV's scalar score is monotonically related to structural error severity
across a wide range of error types and severities. Binary equivalence
metrics cannot show this — they collapse partial/overweak/overstrong all
to "not equivalent." Surface metrics (BLEU, BERTScore) saturate on this
task. ρ = 0.8543 with a 95% CI that excludes 0.81 is a strong gradedness
result.

### What it doesn't prove

That SIV's score is *useful* downstream. Gradedness is necessary but not
sufficient for actionability — that's C2's job.

### Reviewer risks and replies

- *"Why three views (a/b/c)?"* The principled filter is pre-statable
  (Vampire bidirectional entailment). All three views are reported for
  transparency. (a) is the headline because the filter identifies cases
  where v1 candidates were misaligned with v2 gold, and regenerating fixes
  that without changing the protocol.
- *"Did you cherry-pick the filter to inflate ρ?"* No — the filter is
  Vampire equivalence, applied uniformly. The fact that it picks out
  exactly the cases the forensics flagged is evidence it's measuring the
  right thing.
- *"What if regeneration is gaming?"* (c) shows ρ on the equivalent-only
  subset (no regeneration anywhere), which is 0.8583. The headline doesn't
  depend on the regenerated cases.

## 5. Experiment C1 — Per-aspect diagnostic structure (intrinsic)

### What it is

Confusion matrix: candidate's actual error type (overstrong / partial /
overweak / gibberish / specific operator perturbation) vs. SIV's
failed-probe signature. Tests whether different error types produce
distinguishable failure signatures.

### Method

- **Inputs:** Same candidate pool as Exp B + Exp A perturbations.
- **For each candidate:** record which probe categories failed (positive
  sub-entailment, contrastive arg_swap, contrastive negation_drop, etc.).
- **Construction:** confusion matrix rows = actual error type, columns =
  dominant failure signature.
- **Statistic:** macro-F1 across error types, plus diagonal vs.
  off-diagonal mass ratio.

### Numbers (expected from re-score)

Pre-registered acceptance: macro-F1 ≥ 0.65 (lock this number before
re-scoring; adjust only if pilot reveals it's the wrong scale).

v1 confusion matrix exists from old Exp 4. Re-score against v2 suites
should preserve or improve the structure since v2 suites are derived from
faithful gold rather than LLM-extracted canonical.

### What it proves

SIV's diagnostic output isn't just a scalar — different failure modes
produce different probe-level signatures. This is the bridge to C2: the
per-probe trace contains *structured* information about how a translation
is wrong, not just *how much* it's wrong.

### What it doesn't prove

That a downstream consumer (LLM, human) can actually use the structure.
That's C2.

### Why C1 matters even though it's a re-score

Without C1, a reviewer can argue any C2 gain comes from "having feedback
at all" rather than "having structured per-aspect feedback." C1 establishes
that the structure exists; C2 establishes the structure is useful.

## 6. Experiment C2 — LLM self-correction with per-probe feedback (HEADLINE EXTRINSIC)

This is the most important experiment in the paper and the one most likely
to fail if designed badly. Treat the design as itself a research problem.

### What it is (the claim)

Compare LLM self-correction success rates under different feedback regimes:
score-only vs. score + per-probe diagnostic. If SIV's per-probe trace is
actionable, the diagnostic condition wins.

### What we are NOT claiming

We are NOT claiming "SIV makes LLMs better at FOL translation." That's a
bigger claim than the experiment supports and reviewers will smell the
overreach. **The quantity being measured is the information content of the
per-probe trace, not the absolute capability of LLM-with-SIV.**

### Three real risks

1. **LLMs too smart.** Frontier models may one-shot the correction without
   feedback, leaving no headroom. Or they may saturate at high accuracy
   under both conditions.
2. **SIV guidance misleads.** Models often over-correct on specific
   feedback, fixating on the flagged aspect at the expense of the rest.
3. **Information leakage.** A probe trace like "positive probe
   ∃x.Loves(x, alice) failed" nearly hands the LLM the fix, conflating
   "diagnostic structure" with "answer key."

Each risk has a design mitigation. None are solvable in the abstract; all
require pilot data.

### Pre-C2 pilots (required, run before locking design)

Run these *before* committing to the main C2 design. They are not the
experiment; they decide what the experiment looks like.

#### Pilot 1 — Baseline correction rate calibration

**Question:** Across candidate sources, what's the no-feedback correction
rate? We need it in the 20–50% band for signal to be detectable.

**Method:**
- Sample 20–30 broken candidates per source (Exp B partial/overweak/
  overstrong pool, Exp A operator-perturbed gold, broken FOLIO gold).
- For each, prompt frontier model (e.g., GPT-4o or Claude Sonnet) with NL
  + broken candidate + "produce a corrected FOL." No SIV info.
- Score corrections via Vampire equivalence to gold.

**Decision rule:**
- If frontier baseline correction > 60% on a source → that source is too
  easy; either drop it or harden candidates.
- If < 10% → too hard; drop.
- If 20–50% → keep.
- Lock the source mix and difficulty before main run.

#### Pilot 2 — Leakage probe (granularity of diagnostic feedback)

**Question:** At what granularity does per-probe feedback inform direction
without leaking content?

**Method:** On 20 candidates from pilot 1's "kept" set, run three diagnostic
granularities:
- **Probe formulas** ("probe `∃x.Loves(x, alice)` failed").
- **Probe categories** ("a positive sub-entailment probe failed; an
  arg_swap contrastive probe did not fail when it should have").
- **Structural descriptions** ("a quantifier-scope-related probe failed").

Score correction success at each granularity.

**Decision rule:** Pick the *coarsest* granularity that still produces a
meaningful gain over score-only. If all three granularities give the same
gain, use the coarsest (least leakage risk). If only the finest works, the
result is suspect — likely leaking the answer.

#### Pilot 3 — Over-correction check

**Question:** Does diagnostic feedback cause regressions (partially correct
candidate becomes worse)?

**Method:** On 20 candidates that are partial (close to correct, missing
one element), run score-only vs. score + chosen-granularity diagnostic.
Classify outputs as: improved / unchanged / partially improved / regressed.

**Decision rule:**
- If regression rate under diagnostic is significantly higher than
  score-only, the experiment must report this honestly. Add regression
  rate as a pre-registered secondary outcome in the main run.
- Don't suppress the finding. "Diagnostic feedback helps on average but
  causes over-correction on partial candidates" is a real and useful
  result.

#### Pilot 4 — Model scaling sanity check

**Question:** Does the diagnostic effect appear across model sizes, or
only at one scale?

**Method:** Run pilots 1 and 3 with three models (frontier, mid-tier,
open-weights ~7–8B). 10 candidates each, just to detect order-of-magnitude
differences in baseline and diagnostic gain.

**Decision rule:**
- If frontier saturates and small model is hopeless → focus the main run
  on mid-tier where signal is detectable.
- If all three show signal → run all three in main experiment.

### Main experiment design (locked after pilots)

The following is the *target* design; pilot results may push specific
parameters in either direction.

**Models:** Three. One frontier (GPT-4o or Claude Sonnet — pick one and
lock the version+date), one mid-tier (4o-mini or Haiku), one open-weights
(Llama-3.1-8B-Instruct or Qwen2.5-7B). Temperature 0. Three seeds per cell
(seed = prompt phrasing variant + candidate ordering).

**Candidates:** 60–100 broken FOLs, locked from pilots. Mix:
- 40% Exp B partial/overweak/overstrong pool (held out from B's reported set).
- 40% Exp A operator-perturbed gold (held out from A's reported set).
- 20% hand-corrupted from broken FOLIO gold (using portion held out from
  the 30 hand-corrections used in D).

**Conditions (5):**
1. **No feedback.** NL + broken candidate. Anchors what "any feedback at
   all" buys you.
2. **Score-only.** Adds "this candidate scored 0.34 against gold."
3. **Score + binary per-probe.** Adds "3 of 5 positive probes passed; 1
   of 4 contrastive probes failed when it should have."
4. **Score + per-probe diagnostic.** Adds the granularity chosen by Pilot
   2 (most likely structural-level descriptions).
5. **Shuffled-trace ablation.** Same as (4) but probe-result mappings
   randomly permuted within premise. Critical for distinguishing
   "structured feedback helps" from "any feedback helps."

Optional but strongly recommended: **(6) baseline-metric feedback** — an
analogous trace from BLEU or equivalence-result. Even one alternative
shows SIV-helps-*more*, not just SIV-helps.

**Prompt scaffolding:** Identical across all conditions. Same task
description, same format for NL premise, same format for candidate. Only
the feedback block differs. Both prompts in appendix.

**Few-shot vs. zero-shot:** Zero-shot. Few-shot introduces a degree of
freedom (which exemplars? same across conditions?) that's costly to
control cleanly.

**Outcome metrics (pre-registered):**
- *Primary:* logical-equivalence success rate (Vampire-checked, candidate
  ↔ gold).
- *Secondary 1:* regression rate (was partial → became worse).
- *Secondary 2:* partial-improvement rate (more probes pass but not
  equivalent).

Critically, success is measured against gold *equivalence*, not against
probe pass-rate. This prevents Goodhart-style gaming.

**Statistical test (pre-registered):**
- Per-model paired bootstrap on candidate-level success rate.
- Primary comparison: condition (4) per-probe diagnostic vs. condition (2)
  score-only.
- Secondary comparisons: (4) vs. (5) shuffled-trace, and (4) vs. (3)
  binary per-probe.
- Holm correction across secondaries within each model.
- Primary acceptance: p < 0.05 with Δ ≥ 0.05 absolute success rate
  improvement. Effect size matters; bare significance on a 60-candidate
  set is not enough.

**Sample size justification:** Compute from pilot effect size. If pilot
shows Δ ≈ 0.10 between conditions, n=60 per cell gives ~80% power for the
primary comparison at α=0.05. If pilot Δ < 0.05, the experiment may not
be feasible at planned scale and we revisit.

### Pre-registered failure modes

Decide *now* what we do under each outcome:

- **Diagnostic > score-only, Δ ≥ 0.05, p < 0.05 across ≥2 models.** Headline
  result: "per-probe diagnostic feedback improves correction success from
  X% to Y% (Δ = Z) across model scales." Strongest version of the paper.
- **Diagnostic > score-only on mid-tier only.** Honest report: "the
  diagnostic effect is most pronounced where models lack strong FOL priors;
  frontier models saturate and small models hit a floor." This is a real
  finding.
- **Diagnostic ≈ score-only.** Honest report: "per-probe diagnostic output,
  while informative for human inspection (C1), is redundant with
  score-level information for current LLM self-correction." Diagnostic
  axis becomes "informative for analysis" rather than "actionable
  downstream." Weaker but still defensible. Paper still has B and C1.
- **Diagnostic < score-only.** Report it. Investigate over-correction as
  the cause; likely a paragraph in limitations.
- **Shuffled-trace ≈ structured trace.** Means structure isn't the
  source of the gain. Significantly weakens the per-aspect claim. Honest
  reporting required.

The dishonest moves are: filtering candidates after seeing results, changing
the metric, dropping a model. Pre-registration prevents these.

### What it proves (if it works)

SIV's per-probe diagnostic output is not just descriptive but *actionable*:
giving an LLM the per-aspect failure signature improves its ability to fix
its own broken translations beyond what the scalar score alone provides,
and beyond what randomly-permuted trace structure provides. This is the
strongest possible evidence that SIV's design choices (per-aspect
decomposition, prover-grounded probes) translate into downstream value.

### What it doesn't prove

That SIV is the best self-correction signal possible. Other diagnostic
metrics could do better; we just show SIV does better than score-only and
than its own shuffled trace.

## 7. Experiment D — Gold-quality detection (supporting + artifact)

### What it is (re-framed)

Old framing (pre-pivot): "SIV is reference-free, so it doesn't inherit
gold corruption." That asymmetry was binary — SIV had it, others didn't.

New framing (post-pivot): SIV's machinery *surfaces* gold corruption.
Test-suite generation fails on broken gold; the parser rejects unparseable
gold; round-trip equivalence checks flag inconsistent gold. Other metrics
(BLEU, BERTScore, MALLS-LE, equivalence) silently score against broken
references.

### Method

- **Inputs:** 30 hand-corrected FOLIO gold premises (existing artifact)
  plus the broken FOLIO subset (37% of FOLIO failed Vampire entailment).
- **Run 1 — detection rate:** for each broken premise, does SIV's pipeline
  flag it (parser rejection / round-trip failure / suite-generation
  refusal) before scoring? Compare against silent-acceptance rate of
  baseline metrics.
- **Run 2 — score sensitivity:** score the corrected vs. broken gold with
  v2 suites. Show corrected scores higher.

### Numbers

37% gold corruption rate is the headline existing finding. Detection rate
of SIV's pipeline on the broken set: TBD from re-run, expected ≥80%.

### What it proves

SIV's gold-derived test suites are *checkable* — when gold is broken, SIV
either refuses to generate a suite or surfaces the inconsistency, while
reference-based metrics swallow broken gold and produce silent
high-confidence scores. This is a different and weaker contribution than
the old reference-freedom claim, but it's real and useful.

### What it doesn't prove

That SIV is immune to gold corruption (it isn't, under Approach C). Just
that broken gold is detectable rather than silent.

### Why D is supporting, not a pillar

Under the new framing the asymmetry between SIV and reference-based
metrics is much smaller. Detection-of-broken-gold is genuinely useful but
it's a corollary of the prover-grounded design, not a thesis. The paper
mentions this finding, releases the 30 hand-corrections as artifact, and
moves on.

### Possible cross-use with C2

The 30 hand-corrected premises are a candidate test-bed for C2: real
broken-in-the-wild FOLs paired with known-good corrections. Use a portion
held out from D's reported runs.

## 9. Timeline and sequencing

Three weeks to EMNLP. Order of operations:

1. **Stage 5** — re-score Exp A with v2 suites against ±5pp acceptance
   gate. Closes A. ~1 day.
2. **Stage 1 cleanup** — round-trip-failure soundness check, apostrophe
   encoding, free-indvar reconciliation, XOR audit. Re-verify 99.7%
   round-trip equivalence after. ~2 days.
3. **C1 re-score** — confusion matrix on v2 suites against macro-F1 ≥ 0.65
   pre-registered acceptance. ~1 day.
4. **D re-frame and run** — detection rate of broken gold; score sensitivity
   on corrected vs. broken. ~1 day.
5. **C2 pilots 1–4** — baseline calibration, leakage probe,
   over-correction check, model scaling. Decide candidate sources,
   diagnostic granularity, model lineup. ~3–4 days.
6. **C2 pre-registration document.** Lock everything before main run.
   ~half a day.
7. **C2 main run** — 5 conditions × 3 models × locked candidate set,
   3 seeds. ~1 week including analysis.
8. **Write-up.** Concurrent with steps 4–7 where possible.

If timeline forces cuts, the cleanest reduced version of C2 is: 2 models
(frontier + mid-tier), 50 candidates, 3 conditions (score-only, per-probe
diagnostic, shuffled-trace). Drop the open-weights model and the binary
per-probe condition. Still publishable. Don't drop the shuffled-trace
ablation under any circumstances — it's the load-bearing control.

## 11. Critical reviewer attacks and prepared replies

| Attack | Reply |
|--------|-------|
| "You give up reference-free; the paper is now narrower" | Yes, intentionally. The pivot was forced by a circularity attack and the new framing (gold-derived, prover-grounded, graded, per-aspect) is sound where the old one was not. Better to have a narrower, defensible claim than a broader unsound one. |
| "B's headline depends on regenerating 13 of 35 cases" | Three views reported. (c) equivalent-only subset (n=22) is at 0.8583, no regeneration anywhere. The headline doesn't depend on the regenerated cases; regeneration is reported because excluding the non-equivalent cases entirely would be cherry-picking. |
| "C2's gain might be from 'feedback in general,' not from SIV specifically" | Shuffled-trace ablation (condition 5) addresses this directly. If structured-trace > shuffled-trace, structure is the source of the gain. |
| "C2's diagnostic might leak the answer" | Pilot 2 (granularity selection) explicitly tests this. We pick the coarsest granularity that still produces a gain. Fine-grained probe formulas are not used in the main run unless coarser fails — and if only fine-grained works, we report the result with appropriate caveat about leakage risk. |
| "Why not test SIV on a benchmark beyond FOLIO?" | Out of scope. The paper is a metric for FOL translation, validated on FOLIO. Generalization to other FOL datasets is future work and we say so explicitly. |
| "Closest precedent (Zhong/Yu/Klein 2020) does the same conceptual move for SQL" | Cited prominently. SIV differs in: (i) FOL not SQL, (ii) graded scoring not binary denotation, (iii) prover-checked probes not query execution. Each difference is non-trivial. |
| "Thatikonda et al. 2025 already audits FOL metrics" | Their work *audits* existing metrics with perturbation analysis. SIV *builds* a new metric using prover-checked probes. Complementary, not competing. |
| "EPR is reference-free without your circularity" | EPR uses entailment labels (EntailmentBank, eQASC, e-SNLI), so it's "reference-free" only in the no-reference-FOL sense. Different problem setup. We discuss this distinction in related work. |
| "Why ρ acceptance ≥0.81 not ≥0.75?" | Pre-registered before v2 re-score. Tightening is honest; loosening would not be. The principled-filter approach plus regeneration is what carried (a) over the gate; without it we'd have stopped at 0.7797. |
| "What if all your numbers come from LLM-generated candidates?" | Mix is intentional: B's candidates are LLM-generated under structured prompts (overstrong/partial/etc.); A's are deterministic operator perturbations on gold; C2 mixes both plus broken-in-the-wild gold. Cross-source consistency is itself evidence the metric isn't just memorizing one generator's distribution. |
