## Probe Families in SIV v2

The SIV v2 test suite produces two types of probes:

1. **Positive probes** (sub-entailment tests): Formulas that should be entailed by any correct translation. Generated by decomposing the gold formula into sub-entailments per §6.4. No `mutation_kind` label.

2. **Contrastive probes** (mutation tests): Mutated formulas that should be inconsistent with any correct translation. Generated by 6 operators (§6.5), each labeled with `mutation_kind`:
   - `negate_atom` — flip negation on an atomic predicate
   - `swap_binary_args` — swap arguments in a binary predicate
   - `flip_quantifier` — flip ∀↔∃
   - `drop_restrictor_conjunct` — drop a conjunct from the restrictor
   - `flip_connective` — flip and↔or, implies↔iff, reverse implication
   - `replace_subformula_with_negation` — wrap a subformula in negation

## Category Vocabulary (6 categories)

| Category | Definition | Mapped From |
|----------|------------|-------------|
| **argument-order** | Wrong argument positions in predicates | `swap_binary_args` |
| **quantifier-scope** | Wrong quantifier type/order/scope | `flip_quantifier` |
| **restrictor-content** | Missing/added/wrong restrictor conjuncts | `drop_restrictor_conjunct` |
| **polarity** | Missing/extra negation on atoms or subformulas | `negate_atom`, `replace_subformula_with_negation` |
| **connective-polarity** | Wrong connective: and/or, implies/iff, direction | `flip_connective` |
| **content-gap** | Missing semantic content (positive probe failure) | positive sub-entailment failures |

## Mapping Logic

A **category is detected** for a candidate when:
- For contrastive-based categories: a contrastive probe of the corresponding `mutation_kind` is **consistent** with the candidate (should be inconsistent with correct translation, so consistency = error detected).
- For content-gap: a positive sub-entailment probe is **not entailed** by the candidate.

## Important Notes

1. `polarity` combines two operators (`negate_atom` + `replace_subformula_with_negation`) because both test negation errors at different structural levels. The feedback message will say "polarity error detected" without distinguishing atom-level from subformula-level.

2. `connective-polarity` is kept separate from `polarity` because the error type ("wrong connective") is cognitively distinct from "wrong negation" even though both involve truth-value flipping. An LLM hearing "connective-polarity error" should check and/or/implies/iff choices, while "polarity error" should check negation signs.

3. `content-gap` is the only category derived from positive probes. It captures structural incompleteness (missing predicates, omitted conjuncts) rather than structural incorrectness.
